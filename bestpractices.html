Structure and State

  • Organize your code in .example blocks to separate functionalities.
  • Initialize state with <init> for clarity and reusability.
<div class="example">
  <init>
    state.name = '';
    state.count = 0;
  </init>
  ...
</div>

Main Directives

  • @model: Two-way binding on inputs/texts
  • @text: Inserts reactive text
  • @if: Conditional (show/hide)
  • @show: Dynamically changes display
  • @for: Iterates over arrays/objects
  • @key: Key for dynamic lists
  • @click: Handles click events
  • @switch, @case, @default: Declarative switch-case
  • @watch: Run code on variable change
  • @fetch: Load data from APIs
  • @result: Where to save the result of fetch/map/filter/reduce
  • @log: Quick debug
  • @validate: Input validation
  • @class: Dynamic classes
  • @style: Dynamic styles
  • @component: Use custom components
  • @source, @map, @filter, @reduce, @initial: Functional on arrays
  • @animate: Animations on elements
  • @page, @link: SPA routing
  • @set: Set values on events
  • @state: Show global state

Sub-directives and Events

  • @click.once: Click only the first time
  • @input.debounce.500: Debounce on input
  • @model:input, @model:change: Model only on input/change
  • @fetch:click: Fetch only on event
  • @set:click, @set:change: Set on event
  • @class:focus, @class:input, @class:hover, @class:click: Dynamic classes on event
  • @text:hover, @text:click: Change text on event

Practical Examples

  • Reactivity: <input @model="name"> <span @text="name"></span>
  • Conditional: <div @if="count > 0">Positive!</div>
  • List: <li @for="item in items" @key="item.id">...</li>
  • Fetch: <div @fetch="url" @result="data"></div>
  • Validation: <input @validate="required|email">
  • Components: <div @component="user-card" ...></div>
  • Animation: <div @animate="fadeIn"></div>
  • SPA: <div @page=""></div>, <a @link="/home">Home</a>
  • Sub-directives: <button @fetch:click="url">Load</button>

General Tips

  • Comment and document complex examples.
  • Use simple and accessible CSS classes (.primary, .red-border).
  • Progressive enhancement: add directives even to legacy HTML.
  • Use @state and @log for debugging.
  • See examples.html for all real use cases.
  • Variable initialization: you can write either <init>state.var = true;</init> (to modify the global state) or <init>var = true;</init> (for variables local to the context). Prefer state. for variables shared between multiple components or examples.
  • Fetch without @result: if you omit @result in a fetch, the result is automatically placed in state.result (e.g. <div @fetch="url"></div> → state.result).
  • Consistency: always use the same naming conventions for variables and state keys to avoid confusion.
  • Accessibility: add aria-label and descriptive placeholders to inputs for better UX.
