Core Directives

@model
(Two-way binding) Creates a bidirectional data binding between an input element and a state property. When the user types or changes the input value, the state is automatically updated. When the state changes programmatically, the input value is automatically synchronized. This directive does not have sub-directives - it automatically handles all input events appropriately.
Use for: form inputs, textareas, selects, checkboxes, radio buttons. Always creates the state property if it doesn't exist. Works with any element that has a 'value' property.
@click
(Event handler) Executes JavaScript code when the element is clicked. The expression has full access to the reactive state object and can perform complex operations, function calls, or state mutations. Supports multiple statements separated by semicolons or commas.
Use for: buttons, links, any clickable elements. Can execute multiple statements: count++; name='clicked'. Automatically triggers re-render after execution.
@for
(Loop) Renders the element for each item in an array or object. Supports two syntaxes: item in items for value-only iteration, or index, item in items for index and value. When using the two-variable syntax, the first variable is always the index (starting from 0), the second is the item value. Automatically converts objects to arrays using Object.values(). Provides special context variables: $index, $originalIndex, and $arrayName.
Use for: lists, tables, repeated UI patterns. Always pair with @key for optimal performance and correct updates. Works with filtered arrays and maintains original array references for complex operations.

    (Conditional rendering) Conditionally renders the element in the DOM only when the expression evaluates to truthy. When false, the element is completely removed from the DOM tree, including all event listeners and child components. This is different from @show which only toggles visibility.
    Use for: conditional components, error messages, optional content. Best for content that should not exist in DOM when hidden. More efficient for large content blocks that are rarely shown.

    @show
    (Visibility toggle) Controls element visibility by setting display: none when the condition is false, but keeps the element in the DOM tree. Unlike @if, the element continues to exist and maintains its state, event listeners, and child components. Better for frequently toggled content.
    Use for: dropdown menus, modals, frequently toggled content. Preserves element state and is more efficient for content that toggles often. Use instead of @if when you need to maintain component state. 
    Visible
    @hide
    (Inverse visibility) Hides the element by setting display: none when the condition is true - the opposite behavior of @show. The element remains in the DOM tree but becomes invisible when the condition evaluates to truthy. Useful for "hide when" logic instead of "show when".
    Use for: hiding elements based on conditions, error states, loading overlays. Semantically clearer than @show="!condition" when you want to express hiding logic directly.

    @fetch @result
    (HTTP requests) Performs asynchronous HTTP requests to the specified URL. The URL can be a string literal, a state variable, or a complex expression with interpolation (e.g., /api/users/{userId}). Always pair with @result to store the response data. Supports sub-directives for event-triggered requests like @fetch:click, @fetch:hover, @fetch:change. Automatically handles JSON parsing and error states.
    Use for: API calls, dynamic data loading, form submissions. The @result variable will contain the parsed JSON response. Use @fetch:click for user-triggered requests. Supports URL interpolation with curly braces.
    @validate
    (Input validation) Validates input values using 8 built-in validators or custom regex patterns. Default validators: required (non-empty), email (RFC standard), phone (international format), password (8+ chars, uppercase, lowercase, number, symbol) Custom patterns: Use regex:pattern or direct regex like ^[A-Z]\d{3}$. Multiple rules separated by commas. Results stored in global _validate object. Requires @model for field binding.
    Use for: form validation, user input checking, data integrity. Check _validate.fieldName for boolean validation status. Real-time validation on input/blur events. Combine multiple validators: @validate="required,email,minLength:5".
    @page @link
    (SPA routing system) Creates a single-page application routing system. @page marks an element as a route destination - it will only be shown when _currentPage matches its value. @link creates navigation links that update _currentPage without browser reload. The special value @page="" creates a default/home page. Navigation preserves all application state and provides instant transitions.
    Use for: multi-view applications, dashboards, wizards. Access current route via _currentPage variable. Use @link="pageName" on buttons, links, or any clickable element. Automatically handles browser history and URL synchronization.

    @component @src
    (Reusable components) Loads and renders external HTML files as reusable components. @src specifies the relative or absolute file path. Components inherit the parent's state and context, making them truly reactive. Supports dynamic loading based on state variables. Can be combined with @page for route-specific components and @if for conditional loading. Loaded content is cached for performance.
    Use for: modular UI architecture, shared components, layouts. Keep components focused and small. Use path like ./header.html or components/modal.html. Components can contain their own directives and sub-components.
    @state
    (Debug inspector) Displays the entire reactive state object as formatted JSON for debugging and development. Updates in real-time as state changes, showing current values of all state properties. Includes syntax highlighting and proper indentation for readability. Very useful during development to monitor state mutations.
    Use for: development debugging, state inspection, learning. Remove from production builds. Shows the complete state tree including nested objects and arrays. Updates immediately when any state property changes.
    @log
    (Directive logger) Shows a comprehensive log of all directive activities on the current element. Displays directive execution, value changes, event triggers, and performance metrics with color-coded output. Tracks sub-directives and provides detailed debugging information including execution timing and data flow. Essential for understanding directive behavior and troubleshooting complex interactions.
    Use for: directive debugging, learning framework behavior, performance analysis. Shows real-time directive execution with timestamps. Great for understanding the order of directive processing and data flow.
    @class
    (Dynamic CSS classes) Dynamically applies CSS classes based on object notation or string expressions. Object syntax: {className: condition} applies the class when condition is truthy. String syntax evaluates to class names. Supports sub-directives for event-based class changes like @class:focus, @class:hover, @class:click. Multiple classes can be applied simultaneously.
    Use for: conditional styling, interactive states, theme switching. Object syntax: {active: isActive, disabled: !enabled}. String syntax: "class1 class2" or computed expressions. Works great with CSS transitions.
    @style
    (Dynamic inline styles) Dynamically sets inline CSS styles using object notation. Each key-value pair represents a CSS property and its value. Supports camelCase (backgroundColor) or kebab-case ('background-color') properties. Values can be computed expressions, state variables, or static values. Automatically handles units for numeric values where appropriate.
    Use for: dynamic styling, animations, responsive layouts. Example: {color: textColor, fontSize: size + 'px', display: showElement ? 'block' : 'none'}. Great for user-customizable interfaces and dynamic themes.
    @set
    (State initialization) Sets initial values for state properties only on the first render, identical to blocks but element-scoped. Use syntax @set="variable=value" for initialization. This is a one-time assignment that runs only when the element is first mounted - it does NOT update on subsequent renders or state changes. Unlike other directives, @set does not support sub-directives or event-based execution.
    Use for: component initialization, default values, element-specific state setup. Runs once on mount only. For event-driven updates, use @click with assignments or other event directives.

    @key
    (Unique key) Provides a unique identifier for elements in @for loops to optimize DOM updates.
    Use for: lists, tables. Always use with @for for best performance.

    @animate
    (Animation) Applies a CSS animation or transition to the element when it enters the DOM. The animation name should match a CSS animation defined in your stylesheet.
    Use for: visual effects, smooth transitions, UI polish. Define the animation in CSS first, then reference it by name. Great for fade-ins, slide effects, and attention-grabbing animations.

    Use for: reactive side effects, logging, computed values, API calls on data change. Example:

    @hover
    (Event handler) Executes code when the element is hovered. Perfect for showing tooltips, updating status variables, or triggering visual feedback. Can update multiple state variables in a single expression.
    Use for: tooltip triggers, hover states, preview loading. Example: showTooltip = true; previewData = item. Combine with @class:hover for CSS styling.

    @text
    (Dynamic text) Sets the text content of the element to the value of an expression or variable. For single expressions, displays the evaluated result. For multiple assignments (detected automatically), executes the code instead of displaying text. Supports sub-directives for event-based text updates.
    Use for: dynamic labels, counters, computed text. Prefer over {{ }} interpolation for better performance. Handles null/undefined gracefully.
    @input
    (Input event handler) Executes code when the element's input event is triggered. Perfect for real-time search, live validation, and instant updates. Access the current value via event.target.value or bind directly to state variables.
    Use for: live search filters, character counters, instant validation. Example: search = event.target.value; results = items.filter(i => i.includes(search)). Updates on every keystroke.
    @change
    (Change event handler) Executes code when the element's change event is triggered. Fires when an input loses focus after its value has been changed, or when select/checkbox values change. Perfect for final validation and processing.
    Use for: form validation, data processing, state updates. Example: isValid = event.target.value.length > 3; if(isValid) saveData(). Fires only when user finishes editing.
    @focus
    (Focus event handler) Executes code when the element receives focus. Perfect for showing help text, clearing error messages, or highlighting form sections. Can update multiple state variables to enhance user experience.
    Use for: help text display, error clearing, UI highlighting. Example: helpVisible = true; errors = ''; activeField = 'username'. Great for dynamic form behavior.
    @blur
    (Blur event handler) Executes code when the element loses focus. Perfect for hiding help text, saving form data, or capturing the final value for processing. Opposite behavior of @focus.
    Use for: hide help text, save data, capture final values. Example: helpVisible = false; lastValue = event.target.value; saveProgress(). Triggers when user moves away from field.

    Sub-directives & Events

    @fetch:click
    (Fetch on click) Executes a fetch request only when the element is clicked. Combines the power of @click and @fetch for user-initiated API calls. Use with @result to store the response data.
    Use for: load-on-demand, user-triggered API calls. Perfect for buttons that fetch data when clicked.
    @text:click
    (Text on click) Updates the element's text content when clicked. Perfect for toggle buttons, status indicators, or interactive labels that change on user interaction.
    Use for: toggle buttons, status changes, interactive labels. Great for buttons that show different states or confirmation messages when clicked.
    @text:hover
    (Text on hover) Updates the element's text content only when the user hovers over it. Reverts to original text when hover ends. Perfect for interactive tooltips or dynamic labels.
    Use for: hover tooltips, interactive labels, context-sensitive help. Great for buttons that show additional info on hover.

    @class:focus
    (Class on focus) Applies CSS classes only when the element receives focus. Removes the classes when focus is lost. Perfect for form styling and accessibility feedback.
    Use for: form input styling, focus indicators, accessibility. Great for highlighting active form fields or providing visual focus feedback.

    Special Variables

    state: Global reactive state object.
    _validate: Object with input validation results.
    _currentPage: Active page for SPA routing.

    Best Practices & Tips

    • Use state for all reactive variables and shared data between components.
    • Prefer @model for two-way binding with form inputs over manual event handling.
    • Use @key with @for loops for optimal performance and correct DOM updates.
    • For validation, combine @validate with @model and check _validate for results.
    • Use @fetch with @result for async data loading - they work together.
    • For SPA routing, use @page to define routes and @link for navigation.
    • Keep components small and focused - use @component with @src for reusability.
    • Use @watch for side effects and computed values, not for heavy logic.
    • Sub-directives (like @fetch:click) combine base functionality with event triggers.
    • Use @set for one-time initialization only - it works like blocks.
    • For debugging, use @state and @log to inspect state and directive flow.
    • Most directives can be freely combined on any element for complex behavior.
    • Use @animate for smooth transitions - define CSS animations first.
    • Prefer @text over {{ }} interpolation for better performance.
    • Use @if to remove elements from DOM, @show/@hide to toggle visibility.
